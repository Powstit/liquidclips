# Liquid Clips v0.8.0 — Interface Upgrade Specification

> For Claude HTML demo build. Kimi will implement into the Tauri/React codebase after approval.

## Design Philosophy

**Seamless. No modal prison.** Every edit, reaction, caption, trim, and schedule action happens in-place. The user never leaves the main workspace unless they're managing connected accounts or viewing analytics.

**Dense but breathable.** Information-rich panels with tight spacing (8px grid) but clear visual hierarchy. No random whitespace. Every pixel earns its place.

**Compliment existing DNA.** Fuchsia (`#d946ef`) primary, warm paper backgrounds (`#faf9f7`), ink text (`#1a1a1a`), subtle borders (`rgba(0,0,0,0.06)`). Lucide icons only. No new colors. No new fonts.

## 1. Create + Import Unified Surface

### Current Problem
Two separate flows: "Create" (URL → auto-clip) and "Import" (file upload). They look different, feel different, and confuse users.

### New Design: Single "Add Clips" Surface

**Trigger:** A single floating action button (FAB) in the bottom-right of the grid when no clips exist, OR an "Add Clips ▸" button in the top nav when clips exist.

**Panel (slides up from bottom, 420px height):**

```
┌────────────────────────────────────────┐
│  Add Clips                    [×]      │
├────────────────────────────────────────┤
│  [ URL ]  [ Upload ]  [ Library ]     │  ← Segmented control
├────────────────────────────────────────┤
│                                        │
│  URL Tab:                              │
│  ┌──────────────────────────────────┐  │
│  │ 🌐 Paste YouTube / TikTok URL   │  │
│  └──────────────────────────────────┘  │
│  [ Analyze & Clip ]                    │
│                                        │
│  ─ or ─                                │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │ 📁 Drop files or click to browse │  │
│  └──────────────────────────────────┘  │
│  Supported: MP4, MOV, MKV (up to 2GB) │
│                                        │
└────────────────────────────────────────┘
```

**URL Tab Details:**
- Single input field, full width, 48px height
- Placeholder: "Paste a YouTube, TikTok, or Instagram Reels URL"
- On paste: instant format validation (regex for youtube.com, youtu.be, tiktok.com, instagram.com)
- Valid URL: input border turns fuchsia, button becomes active
- Button: "Analyze & Clip" (primary, full width, 44px height)
- Clicking button: input disabled, button shows spinner + "Analyzing…"
- Progress: indeterminate spinner + text "Reading transcript…" → "Finding hooks…" → "Cutting clips…"
- On complete: panel auto-closes, new clips appear in grid with a "flash" highlight animation
- On error: inline error below input, red text, shake animation on input

**Upload Tab Details:**
- Dropzone: dashed border 2px `border-line`, dashed border-radius 12px, min-height 160px
- Idle state: large Upload icon (Lucide `Upload`, 32px, text-tertiary), text "Drop video files here"
- Drag over: border turns fuchsia solid, background `bg-fuchsia-soft/10`
- Files dropped: list appears with filename + size + progress bar per file
- Progress bar: fuchsia fill, 4px height, rounded
- On complete: thumbnail generated, file row turns green checkmark
- "Import as Clips" button (primary, disabled until all uploads complete)
- Same engine as URL: transcript → hooks → cuts

**Library Tab Details:**
- Grid of previously deleted/archived clips from the library
- Each card: thumbnail + title + "Restore" button
- Search input at top
- Empty state: "No archived clips" with archive icon

**Visual Spec:**
- Panel background: `bg-paper` with `shadow-[0_-8px_32px_rgba(0,0,0,0.12)]`
- Top border radius: 16px (only top corners)
- Backdrop: `bg-ink/20` with blur(4px)
- Animation: slide up 300ms `cubic-bezier(0.16, 1, 0.3, 1)`
- Close: swipe down on mobile, click backdrop or × on desktop

## 2. Clip Grid — Zero Whitespace Rules

### Card Anatomy

```
┌────────────────────────┐
│ ▶️  0:32               │  ← Top-left: play icon + duration
│                        │
│                        │  ← Center: thumbnail (object-cover)
│                        │
│ 💬 "This is the hook"  │  ← Bottom: caption overlay (gradient)
│ ┌────┐┌────┐┌────┐    │  ← Bottom-right: platform badges
│ └────┘└────┘└────┘    │
│ ◻️  🔧  ✂️  🗑️         │  ← Hover: select, edit, split, delete
└────────────────────────┘
```

**Dimensions:**
- Card aspect ratio: 9:16 (vertical clip preview)
- Gap: 12px
- Min width: 180px, max width: 280px
- Grid: auto-fill, responsive

**Zero Whitespace Rules:**
1. Thumbnail fills the ENTIRE card. No padding around the image.
2. Overlays (duration, caption, badges) are absolutely positioned WITHIN the card.
3. Caption text has `max-lines: 2` with `text-shadow: 0 1px 3px rgba(0,0,0,0.8)` for readability.
4. Hover actions appear as an overlay, not pushing content.
5. No card border. No card shadow in idle. Subtle shadow on hover.

**Platform Badges (bottom-right of card):**
- Small circular icons (16px) with platform brand colors
- YouTube: red, TikTok: black/white, Instagram: gradient
- Badge shows connection state:
  - Connected: full color, subtle glow
  - Disconnected: grayscale, 50% opacity
  - Hover disconnected badge: tooltip "Connect Instagram ▸"
- Clicking a disconnected badge opens Ayrshare OAuth inline (popover, not modal)
- Clicking a connected badge toggles scheduling for that platform

**Selection State:**
- Selected card: 2px fuchsia border, `shadow-[var(--glow-sm)]`
- Bulk select: checkbox top-left (always visible on hover, visible when selected)
- Selected count appears in cockpit header

**Backend Flow (what the spec wires up):**

1. `method_thumbnail_save_identity` — persists 3+ face crops to `~/LiquidClips/identity/`
2. `method_thumbnail_save_brand` — persists style preset (from uploaded reference or default)
3. `method_thumbnail_preview_prompt` — zero-cost prompt preview. Shows user the exact prompt before spending money.
4. Auto-metaphor: frontend sends title to GPT-4 text API (or backend `method_build_metaphor`) → returns metaphor string
5. `thumbnail_engine.generate()` — actual image generation with multipart form (reference images attached)
6. `startOverlayBake`-style background job for batches 2–33 (remaining 360)

**Metaphor Generation (the "question"):**

System prompt sent to GPT-4 (text, not image model):

```
You are a YouTube thumbnail concept designer.
Given a video title, create a VISUAL METAPHOR — a concrete scene or object comparison that instantly communicates the video's promise in one image.

Rules:
- Use physical objects, scenes, or contrasts (not abstract concepts)
- Show a "collision of two worlds" — the old way vs the new way
- The metaphor must feature the character physically interacting with the object
- One clear symbolic element, no clutter

Examples:
Title: "Rent or Own?" → Metaphor: "a gleaming house key vs a heavy iron chain"
Title: "I Read 100 Books in a Year" → Metaphor: "a towering stack of books about to topple, person peeking over the top"
Title: "How I Made 365 Thumbnails in 1 Day" → Metaphor: "a wall covered floor-to-ceiling with printed thumbnails, person standing exhausted but triumphant"

Title: "{USER_TITLE}"
Metaphor:
```

Return value is the `metaphor` field passed to `thumbnail_engine.build_prompt()`.

## 4. Platform Connection Flow (Inline)

### Current Problem
User has to leave the clip view to connect accounts. Platform badges on clips don't reflect connection state.

### New Design

**Inline OAuth Popover:**
- Trigger: clicking a disconnected platform badge (on card OR in schedule tab)
- Popover: 320px wide, appears below the badge
- Content:
  ```
  ┌─────────────────────────────┐
  │  Connect Instagram          │
  │                             │
  │  1. Click "Authorize"      │
  │  2. Log in to Instagram    │
  │  3. Return to Liquid Clips │
  │                             │
  │  [Authorize with Ayrshare] │
  └─────────────────────────────┘
  ```
- "Authorize" button: opens Ayrshare OAuth URL in external browser
- Window focus detection: when user returns, app polls Ayrshare for connection state
- On connect: badge animates from grayscale to color, toast "Instagram connected"
- No modal. No route change.

## 5. Animation & Interaction Spec

### Micro-interactions
- **Button press**: scale(0.97) on active, 100ms ease-out
- **Tile selection**: border-color transition 150ms, shadow fade 200ms
- **Panel slide**: transform translateY, 300ms cubic-bezier(0.16, 1, 0.3, 1)
- **Progress bar**: width transition 300ms linear, pulse glow on indeterminate
- **Toast**: slide in from top-right, 400ms, auto-dismiss 4s
- **Card hover**: shadow deepen, action bar fade in 150ms
- **Badge connect**: grayscale→color 300ms, scale(1.1) bounce then settle

### Loading States
- **Button loading**: spinner replaces text, 16px, fuchsia
- **Card processing**: overlay with pulse animation, "Baking…" text
- **Grid loading**: skeleton cards, 9:16 aspect, shimmer animation

### Empty States
- **No clips**: centered illustration (CSS-generated geometric shape) + "Add your first clip" CTA
- **No connection**: platform badge area shows "+" icon, tooltip "Connect to publish"
