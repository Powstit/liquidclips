Connection-by-chip (AyrshareConnectionPanel) vs channel-by-card (ChannelCard). User sees them in two different views and has to learn two interaction models. 690 LOC.
2 P0 Per-row complexity schedule/ChannelCard.tsx L109-303 5 elements per row: dot · name · handle · toggle · (microcopy slot). ~20 elements per card: corner brackets · 40×40 icon container · label/inline-rename · handle · status pill · posts counter · analytics chip · refresh · pause/resume · delete · confirm-delete inline mode · pending_link diagnose panel · result strip · 3 modal spinners · ConfirmDialog for disconnect.
3 P0 Missing toggle UX ChannelCard.tsx — no pill toggle exists The pill toggle (32×18) is the primary affordance: tap-to-route-this-channel. Mockup users immediately understand "this is on, that's off." No toggle anywhere. Pause/resume + delete + refresh actions live in hover row at the bottom. The "is this channel currently routing for me?" question has no clear single visual answer.
4 P1 Connection vs routing mixed AyrshareConnectionPanel.tsx "Connection" = once per platform, lives in Settings. "Routes" = per clip, lives in a modal triggered from the cockpit. Two different concepts, two surfaces. AyrshareConnectionPanel does connection but not routing. ChannelCard does channel-level pause but not per-clip routing. The per-clip routing surface (ClipCard cockpit Routes modal) opens yet a THIRD pattern (lines 463-466 of mockup were never built).
5 P1 Disconnect-by-× AyrshareConnectionPanel.tsx L278-285, L328-346 Toggle off the row's pill. Confirmable inline (microcopy says "paused · resume in Settings"). Tiny × on each chip → opens a destructive-tone ConfirmDialog modal that says "Hide [platform] from publish targets?" Different word ("hide"), different verb, native-feel.
6 P1 Status taxonomy ChannelCard.tsx L18-25 2 states: on (green dot) / warn (amber dot + microcopy). 6 states: active · pending_link · unlinked · error · paused · deleted. Three of those (unlinked, error, pending_link) all surface as red/amber dots with similar reconnect copy. User has to read the microcopy carefully to know which one applies.
7 P2 Corner brackets ChannelCard.tsx L114-117 None — flat 1px border, 6px radius. 4 absolutely-positioned library-card-corner brackets in fuchsia. Adds visual chrome that doesn't pay rent on this surface.
8 P2 Platform icon container ChannelCard.tsx L119-125 None — name field is the platform identifier. 40×40 dark-ink rounded square with outline icon inside. Doubles the visual weight of the row.
9 P2 Hover affordance ChannelCard.tsx L216 (opacity-60 → 100) None — toggle is always visible. Action row at 60% opacity until hover. Discoverable only by hovering — bad for touch and reduced-motion users.
10 P2 Inline rename ChannelCard.tsx L127-148 None — no concept of renaming a channel. Click the channel label → inline input opens. Mockup doesn't have this; the handle is the label.

Recommended path forward

Revert to the ch-row pattern. Delete ChannelCard.tsx (327 lines), simplify AyrshareConnectionPanel.tsx from chip-list to ch-row list (~120 lines after the cut). Build the missing Routes modal from cockpit-handoffs-demo.html L460-471 so the per-clip routing affordance lands using the SAME ch-row component.

Single source of truth for "is this channel routing for me right now?" — the pill toggle. Status complexity collapses from 6 states to 2 (on / warn) with the specific reason in the microcopy slot. Backend behavior (Ayrshare connect/diagnose/pause/delete) is unchanged — these are UI accretion fixes, not protocol changes.

Net diff: ~690 LOC removed, ~250 LOC added for the unified component + new Routes modal. —440 net. One file (ChannelRow.tsx) becomes the single visual contract for every social channel everywhere in the app.
