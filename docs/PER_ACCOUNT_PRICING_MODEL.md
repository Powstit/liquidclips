# Per-Account Pricing Model — Hit $1M with 2,500 Users

**Prepared by:** Kimi (2026-05-31)  
**The insight:** Nobody in creator tooling charges per connected social account. Buffer/Hootsuite charge per channel but bundle them. We charge per account beyond 10 — and clippers managing 20-50 accounts are our most profitable segment.

---

## 1. The "Account" Definition (Critical)

An **account** = one social platform login. Examples:

| User Type | Connected Accounts | Ayrshare Profiles (what we pay) |
|-----------|-----------------|--------------------------------|
| Solo creator | TikTok + IG + YouTube + X + Threads + FB = 6 | 1 profile (same human) |
| Growing creator | Above + LinkedIn + Pinterest + Reddit + Bluesky + backup TikTok = 11 | 1 profile |
| Small agency | 10 client TikTok accounts | 10 profiles (10 different humans) |
| Full agency | 10 clients × 4 platforms each = 40 accounts | 10 profiles (if clients grant access via Ayrshare) |

**Key:** Ayrshare bills per human profile, not per platform. A creator with 10 personal platforms = 1 profile ($2.87). An agency with 10 clients = 10 profiles ($28.70).

**Our pricing charges per connected account regardless.** A creator with 11 platforms pays for 1 extra. An agency with 40 accounts pays for 30 extras. This captures value from both segments.

**Our differentiation:**

- Hitting the 11th account triggers the "I'm a serious creator" moment — they're happy to pay
- Agency users at 25+ accounts feel like they're getting a deal because they remember paying $249/seat at Hootsuite for fewer accounts

## 7. Billing Architecture Changes

### Database: Add account counting

```python
# models.py
class User(Base):
    __tablename__ = "users"
    id = Column(String, primary_key=True)
    tier = Column(String, default="free")
    # NEW:
    included_accounts = Column(Integer, default=5)  # from tier
    extra_accounts = Column(Integer, default=0)     # purchased
    max_accounts = Column(Integer, default=5)       # included + extra

class SocialConnection(Base):
    __tablename__ = "social_connections"
    id = Column(Integer, primary_key=True)
    user_id = Column(String, ForeignKey("users.id"))
    platform = Column(String)  # "tiktok", "instagram", etc.
    handle = Column(String)    # @username
    ayrshare_profile_key = Column(String)
    # NEW: track if this counts toward the limit
    is_active = Column(Boolean, default=True)
    created_at = Column(DateTime, server_default=func.now())
```

### Billing Logic

```python
def calculate_monthly_charge(user: User) -> int:
    base = TIER_BASE_PRICES[user.tier]  # cents
    included = TIER_INCLUDED_ACCOUNTS[user.tier]
    active_count = db.query(SocialConnection).filter_by(
        user_id=user.id, is_active=True
    ).count()
    
    if active_count <= included:
        return base
    
    extra = active_count - included
    overage_rate = TIER_OVERAGE_RATES[user.tier]  # cents per account
    return base + (extra * overage_rate)
```

### Whop Integration

Whop supports tiered pricing but not dynamic overages natively. Options:

1. **Stripe directly** (recommended for overages):
   - Subscription = base tier via Whop
   - Metered billing for extra accounts via Stripe
   - Junior backend calls Stripe API to update subscription item quantity

2. **Whop + manual invoice**:
   - Base tier on Whop
   - At month end, count active accounts
   - Stripe invoice for overages (or credit for next month)

3. **Hybrid — prepaid account packs**:
   - User buys "Account Pack" ($40 for 5 extra = $8/account)
   - Pack lasts until used (no expiration)
   - Simpler billing, no metered complexity

**Recommended: Prepaid account packs.** Users buy packs of 5 extra accounts for $40. Simple, predictable, no metered billing complexity.
